# ADR-001: アプリケーションサービス層の導入

| 項目 | 内容 |
|---|---|
| **ステータス** | 提案 |
| **日付** | 2026-02-25 |
| **決定者** | architect エージェント |
| **関連ファイル** | `components/data_loader.py`, `src/usecase/` (新設) |

---

## コンテキスト

`components/data_loader.py`（1683行）が以下の3つの責務を兼務している。

1. **UI アダプタ**: `@st.cache_data` によるキャッシュ制御、Streamlit セッション状態の管理
2. **ユースケースオーケストレーション**: 複数ドメインサービスの呼び出し順序・エラー処理の制御
3. **ドメイン計算**: リスク指標（ボラティリティ・最大ドローダウン・シャープレシオ等）の直接計算

これらの責務は**変化速度が異なる**にもかかわらず、単一ファイルに混在している。

| 責務 | 変化速度 | 本来の置き場所 |
|---|---|---|
| UI アダプタ / キャッシュ | 高（Streamlit API の変化に追随） | `components/` |
| ユースケースオーケストレーション | 中（ビジネス要件の変化に追随） | アプリサービス層 |
| ドメイン計算 | 低（数値計算ロジックは安定） | `src/core/` |

結果として:

- テストが困難（Streamlit に依存しているため、ユースケースロジックの純粋な unittest が書けない）
- 一部を変更すると無関係な責務に影響が波及する
- `data_loader.py` 単体のコード量が 1683行に達し、可読性・保守性が低下している

---

## 決定

`src/usecase/` ディレクトリを新設し、ユースケース調整ロジックをアプリサービスとして分離する。

### 新設ファイル構成

```
src/
  usecase/
    __init__.py
    snapshot_usecase.py          # ポートフォリオスナップショット取得
    portfolio_history_usecase.py # 資産推移時系列の構築
    health_check_usecase.py      # ポートフォリオ健全性チェック
    economic_news_usecase.py     # 経済ニュース取得・要約
```

### 各ユースケースの責務

| ファイル | 責務 |
|---|---|
| `snapshot_usecase.py` | `portfolio_manager`, `yahoo_client` を呼び出し、スナップショット DTO を返す |
| `portfolio_history_usecase.py` | 資産推移時系列の組み立て・ベンチマーク比較計算 |
| `health_check_usecase.py` | `health_check.py` のオーケストレーション（ドメインロジック自体は触らない） |
| `economic_news_usecase.py` | ニュース取得・LLM 要約・キャッシュ制御（Streamlit 非依存） |

### `components/data_loader.py` の縮小

移行後、`data_loader.py` は以下の役割のみを担う薄いアダプタに縮小する。

```python
# 移行後のイメージ
@st.cache_data(ttl=300)
def get_current_snapshot(...):
    # Why: Streamlit キャッシュはアダプタ層にのみ存在すべき
    return SnapshotUsecase().execute(...)
```

目標行数: 1683行 → 200行程度（約 88% 削減）

---

## 根拠

### ペースレイヤリングの原則

ユースケース層は UI フレームワーク（Streamlit）への依存を**持たない**。
これにより:

- Streamlit を差し替えた場合でも、ユースケースコードは変更不要
- ユースケースロジックを `pytest` で純粋な unittest として実行可能

### テスタビリティの向上

現状: `data_loader.py` は `@st.cache_data` に依存しているため、テスト時に Streamlit のモックが必要。

移行後:

```python
def test_snapshot_usecase():
    # Why: Streamlit 依存がないため、純粋な unittest が書ける
    usecase = SnapshotUsecase(portfolio_manager=mock_pm, yahoo_client=mock_yc)
    result = usecase.execute()
    assert result.total_value_jpy > 0
```

### 保守性

`data_loader.py` の 1683行は、単一責任の原則（SRP）を逸脱している。
アプリサービス層の導入により、変更の局所化が実現する。

---

## 結果

### 正の影響

- `data_loader.py` の大幅縮小（1683行 → 200行程度）
- コアロジックが正しいレイヤー（`src/usecase/`）に再配置される
- ユースケースロジックの純粋な unittest が書けるようになる
- Streamlit 以外の UI（CLI・API サーバ等）への移行コストが下がる

### 負の影響

- 移行コスト: 既存コードのリファクタリングが必要（段階的に実施可能）
- ファイル数の増加: 4つのユースケースファイルが新設される

### リスク

- **`@st.cache_data` の配置**: キャッシュデコレータは Streamlit 依存のため `app.py` または `data_loader.py`（アダプタ側）に残す必要がある。ユースケース内に混入しないよう注意する。
- **段階的移行**: 一括置換よりも、1ユースケースずつ切り出して動作確認しながら進めることを推奨する。
