# ADR-004: キャッシュ無効化による手動データ更新

| 項目 | 内容 |
|---|---|
| **ステータス** | Accepted |
| **日付** | 2026-02-26 |
| **決定者** | architect エージェント |
| **関連ファイル** | `components/data_loader.py`, `src/data/yahoo_client.py` |

---

## コンテキスト

これまで株価データの更新は、ディスクキャッシュの TTL（4時間）に完全に依存していた。

### 問題

- **リアルタイム性の欠如**: 相場急変時やポートフォリオ変更直後に最新データを即座に取得できない。
- **ユーザー体験**: TTL が切れるまで古いデータが表示され続ける。

### 選択肢

1. **コア層に `force_refresh` パラメータを追加する**
   - `yahoo_client.get_price()` に `force_refresh: bool` を追加し、全コールサイトを修正する
   - 影響範囲が広い（コア層インターフェースの変更）

2. **UI 層でキャッシュを直接クリアする**
   - `clear_price_cache()` 関数をコンポーネント層に追加する
   - ディスクキャッシュ削除 + `st.cache_data.clear()` のみで実現可能
   - コア層のインターフェースを変更しない

---

## 決定

**UI 層（`components/data_loader.py`）に `clear_price_cache()` 関数を追加し、ボタン操作でキャッシュを無効化する。**

```python
# 実装概要
def clear_price_cache() -> None:
    # ディスクキャッシュ全削除
    for f in PRICE_CACHE_DIR.glob("*.csv"):
        f.unlink()
    # メモリキャッシュクリア
    st.cache_data.clear()
    # ダッシュボード全体を再描画
    st.rerun()
```

ユーザーが「📥 今すぐ更新」ボタンを押すことで `clear_price_cache()` が呼び出される。

---

## 理由

| 理由 | 説明 |
|---|---|
| **影響範囲の限定** | コア層のインターフェースを変更しないため、影響範囲が UI 層のみに限定される。 |
| **既存パターンの拡張** | `_load_cached_prices` / `_save_prices_cache` の既存パターンをそのまま活用できる。 |
| **Streamlit 公式手段** | `st.cache_data.clear()` は Streamlit 公式のキャッシュ制御 API であり、安定性が高い。 |
| **シンプルな実装** | `force_refresh` パラメータの全伝播と比較して、実装・テストのコストが大幅に低い。 |

---

## 影響

- `components/data_loader.py` に `clear_price_cache()` 関数を 1 つ追加。
- `app.py` に「📥 今すぐ更新」ボタンウィジェットを追加。
- コア層（`src/`）への変更はゼロ。
- キャッシュクリア後は `st.rerun()` によりダッシュボード全体が再描画される。次回株価アクセス時に Yahoo Finance API から再取得される。
