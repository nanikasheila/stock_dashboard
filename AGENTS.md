# AGENTS.md

本プロジェクトで利用可能な AI エージェントの一覧と役割を定義する。

## 概要

`.github/agents/` に5つの専門エージェントが定義されている。
トップレベルエージェント（Copilot Chat）が `runSubagent` で各エージェントを呼び出し、
サブエージェント間の直接呼び出しはできない。

## エージェント一覧

| エージェント | ファイル | 役割 | コード変更 |
|---|---|---|---|
| **developer** | `.github/agents/developer.agent.md` | 実装・デバッグ・テスト | ✅ |
| **reviewer** | `.github/agents/reviewer.agent.md` | コードレビュー・品質・セキュリティ検証 | ❌ |
| **writer** | `.github/agents/writer.agent.md` | ドキュメント・ルール・instructions の作成 | ❌（ドキュメントのみ） |
| **manager** | `.github/agents/manager.agent.md` | 影響分析・タスク分解・計画策定 | ❌ |
| **architect** | `.github/agents/architect.agent.md` | 構造設計・設計判断・非機能要求評価 | ❌ |

## エージェント詳細

### developer（開発エージェント）

実装とテストに特化したエージェント。2つのモードを持つ。

- **実装モード**: コード生成・修正・リファクタリング・バグ修正
- **テストモード**: テストコード作成・実行・カバレッジ確認

変更前に影響範囲を確認し、既存機能を壊さないことを保証する。

### reviewer（レビューエージェント）

コードレビューと品質保証に特化したエージェント。以下の観点でレビューする。

- **設計・構造**: モジュール分割・責務分離・循環依存
- **ロジック・正確性**: 計算ロジック・エッジケース・エラーハンドリング
- **セキュリティ**: シークレット漏洩・インジェクション・権限チェック
- **テスト品質**: カバレッジ・テストパターン・モック戦略

修正指示は構造化して出力する。自らコードは書かない。

### writer（ライターエージェント）

技術文書の作成と保守に特化したエージェント。

- README・ガイド・API ドキュメント
- `.github/` 配下の instructions / rules / skills
- `docs/architecture/` の構造ドキュメント
- 変更履歴・リリースノート

コードの実装やテストは行わない。

### manager（マネージャーエージェント）

タスク管理と計画策定に特化したエージェント。

- 要件分析・タスク分解・影響分析
- 実行計画の策定（どのエージェントに何を依頼するか）
- リスクの洗い出し
- architect へのエスカレーション判断

自らコードは書かず、サブエージェントの呼び出しも行わない。

### architect（アーキテクトエージェント）

システム全体の構造的健全性を評価・設計するエージェント。

- ペースレイヤリングに基づく層の評価
- モジュール配置判断・依存方向の検証
- 非機能要求（パフォーマンス・スケーラビリティ）の評価
- ADR（Architecture Decision Record）の出力

個々のコード品質ではなく、モジュール間の関係性・データの流れといった横断的観点で判断する。

## 連携フロー（大規模タスク）

```
1. manager に影響分析・タスク分解を依頼 → 実行計画を受領
2. エスカレーション該当時、architect に構造評価を依頼
3. manager に計画策定を依頼（architect の判断を含む）
4. developer に実装を依頼
5. reviewer にレビューを依頼
6. レビュー指摘があれば developer に修正指示を転送
7. LGTM まで 5-6 を繰り返す
8. writer にドキュメント更新を依頼（必要な場合）
```

## プロジェクト固有の設定

- モデル: `Claude Sonnet 4.6 (copilot)`（`.github/settings.json` で設定）
- テスト実行: `python -m pytest tests/ -q`
- エントリポイント: `run.py`
